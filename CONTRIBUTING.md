# Contributing to Wallper

First off, thanks for taking the time to contribute! 🎉

## How Can I Contribute?

### 🐛 Reporting Bugs

Before creating a bug report, please check existing [issues](https://github.com/alxndlk/wallper-app/issues) to avoid duplicates.

When filing a bug, include:

- **macOS version** (e.g., Ventura 13.4, Sonoma 14.2)
- **Xcode version** (if building from source)
- **Steps to reproduce** the issue
- **Expected behavior** vs. **actual behavior**
- **Logs or screenshots** if applicable

### 💡 Suggesting Features

Feature requests are welcome! Open an [issue](https://github.com/alxndlk/wallper-app/issues) and describe:

- The problem you're trying to solve
- Your proposed solution
- Any alternatives you've considered

### 🔧 Pull Requests

1. **Fork** the repository
2. **Create** a feature branch from `main`:
   ```bash
   git checkout -b feature/my-awesome-feature
   ```
3. **Make** your changes with clear, concise commits
4. **Test** thoroughly on macOS
5. **Push** to your fork and open a Pull Request

#### PR Guidelines

- Keep changes focused — one feature or fix per PR
- Follow the existing code style (Swift conventions, MARK sections)
- Add documentation comments for public APIs
- Update the README if your change affects it

## 🎨 Code Style

- Use Swift's standard naming conventions
- Organize code with `// MARK: -` sections
- Keep functions focused and reasonably sized
- Add doc comments (`///`) for public interfaces

## 📜 Code of Conduct

This project follows the [Contributor Covenant Code of Conduct](CODE_OF_CONDUCT.md). By participating, you agree to uphold this code.

## 💬 Questions?

Join our [Discord community](https://discord.gg/ksxrdnETuc) for discussions, help, and updates.
